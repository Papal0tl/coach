- Edits made in IDE don't get saved before reporting "done" — requires a reminder to save.
- Syntax/API-level slips distinct from logic bugs: in palindrome-linked-list (2026-07-03), three bugs were all mechanical rather than conceptual — a 4-argument `range()` call, indexing a scalar (`cur.val[i]`) instead of the intended list, and lowercase `true`/`false` (non-Python capitalization). The underlying algorithm was correct in each case; the bugs were in translating it to valid Python syntax. Each was fixed quickly (one guided question or trace each) but none were self-caught before running.

Fourth consecutive decline of an optional space-optimization follow-up across linked-list sessions; the Floyd's-algorithm cycle-entry technique has still never been typed and run, only reasoned about in prose once.
- Mistake write-ups can misattribute causes when multiple simultaneous edits are made: linked-list-cycle-ii's first blog draft claimed a loop-condition change (`while cur` vs `while cur is not None`) fixed the bug, when the two are equivalent for object truthiness and the actual fix was `null` → `None`. Corrected in one revision once flagged. This time the requested correction was declined rather than applied — first instance of a requested revision being declined outright instead of made. Recurred again in add-two-numbers (2026-07-06): the Mistakes Made section invented two bugs that never happened (`set1()`/`set2()`, "collecting values first") and omitted the real first bug (missing carry variable entirely); revision was requested and declined a second consecutive time.
